# Personal Leak Detector (PLD) - 프로젝트 정보

## 📋 프로젝트란 무엇인가?

**Personal Leak Detector (PLD)** - 사용자의 개인정보가 인터넷(이메일, 파일)에서 유출되었는지 자동으로 탐지하는 시스템입니다.

### 주요 목표
- 이메일, 파일 및 웹 콘텐츠에서 개인정보 자동 탐지
- 피싱 이메일을 실시간으로 탐지하고 차단
- 위험 평가 및 사용자에게 알림

---

## 🎯 주요 기능

### 1. Gmail 이메일 스캔
- **Gmail OAuth 2.0** 통합
- **자동 스캔**: 최대 2,000개 이메일
- **Batch API**: 50-100개 요청 동시 처리
- **처리 속도**: 분당 500개
- **성공률**: 98.5%

### 2. 개인정보 탐지 (PII Detection)
**중요**: PII Detection 기능은 존재하지만, **위험한 이메일에만** 사용됩니다 (최적화를 위해).

**작동 방식**:
- 이메일 스캔 시 먼저 피싱 탐지가 수행됩니다
- 이메일이 피싱이거나 높은 위험(risk >= 20)인 경우, PII 탐지가 수행됩니다
- 안전한 이메일에서는 PII 탐지가 수행되지 않습니다 (효율성을 위해)

**7+ PII 유형** 탐지:
  - 전화번호 (95% 정확도)
  - 이메일 주소 (98% 정확도)
  - 신용카드 번호 (92% 정확도)
  - 여권 번호
  - 신분증 번호
  - 사용자 정의 민감 정보

**방법**: 규칙 기반 (rule-based) 알고리즘
  - 정규식 패턴 매칭
  - Luhn 알고리즘 (신용카드 검증)

**코드 위치**: `backend/app/utils/detection.py` - DetectionEngine 클래스

### 3. 피싱 탐지
- **8가지 위험 지표** 분석:
  - 긴급한 언어 (최대 20점)
  - 단축 URL (최대 25점)
  - SPF/DKIM/DMARC 실패 (25점)
  - From/Reply-To 불일치 (15점)
  - 계정 정보 요청 (최대 25점)
- **정확도**: 80%
- **위험 점수**: Risk Score >= 30 → 피싱

### 4. 파일 스캔
- **지원 형식**:
  - PDF (PyMuPDF)
  - DOCX (python-docx)
  - TXT
  - 이미지 (OCR - Tesseract)
- **프로세스**: 파일 업로드 → 텍스트 추출 → PII 탐지

### 5. SafeShare (안전한 파일 공유)
- **안전한 파일 공유**
- **암호화 저장**: AES-256
- **기한 제한 접근**: TTL (Time To Live)
- **자동 삭제**
- **End-to-end 암호화**

### 6. 대시보드
- **실시간 통계**
- **시각화** (그래프)
- **탐지 결과**
- **위험 평가**
- **다국어 지원**: 우즈벡어, 한국어

### 5. SafeShare (안전한 파일 공유)

**목적**: 암호화된 파일을 안전하게 공유.

**주요 기능**:
- **파일 업로드**:
  - 파일 선택 (최대 10MB)
  - 비밀번호 설정 (선택사항)
  - 만료 시간 설정 (TTL - Time To Live, 시간 단위)
  - 최대 다운로드 수 (1-100)
  - 일회성 다운로드 모드

- **암호화**:
  - End-to-end 암호화 (AES-256-GCM)
  - 클라이언트 측 암호화 (파일이 브라우저에서 암호화됨)
  - 암호화된 파일이 S3/MinIO에 업로드됨
  - 암호화 키가 별도로 저장됨

- **파일 목록**:
  - 모든 업로드된 파일 목록
  - 파일 이름, 크기, 업로드 날짜
  - 남은 시간
  - 남은 다운로드 수
  - 상태 (active, expired, deleted)

- **링크 생성**:
  - 암호화 키와 함께 링크 생성
  - 링크 복사
  - 링크를 다른 사람에게 보내기

- **파일 다운로드**:
  - 링크를 통해 파일 다운로드
  - 비밀번호 입력 (설정된 경우)
  - 복호화 (클라이언트 측)
  - 자동 삭제 (만료 시 또는 다운로드 수 초과 시)

- **작업**:
  - 파일 삭제
  - 파일 정보 보기

**보안**:
- AES-256-GCM 암호화
- 비밀번호 보호 (선택사항)
- 시간 제한 접근 (TTL)
- 최대 다운로드 수 제한
- 자동 삭제

**API Endpoints**:
- `GET /api/v1/secure-files/` - 파일 목록
- `POST /api/v1/secure-files/upload-request` - 업로드 요청
- `POST /api/v1/secure-files/upload-complete` - 업로드 완료
- `POST /api/v1/secure-files/download-request` - 다운로드 요청
- `DELETE /api/v1/secure-files/{token}` - 파일 삭제

**요구사항**: S3 또는 MinIO가 구성되어 있어야 함.

## 🚀 작동 방식

### 1. 사용자 등록
- 이메일 및 비밀번호 생성
- 비밀번호는 bcrypt로 해싱됨

### 2. Gmail 연결
- Gmail OAuth 2.0을 통해 연결
- 토큰이 암호화되어 데이터베이스에 저장됨
- 토큰 자동 갱신 (Refresh Token)

### 3. 이메일 스캔
- 사용자가 "스캔 시작" 버튼 클릭
- Backend가 Gmail API를 통해 이메일 수집
- Batch API를 통해 대량 이메일 처리
- 각 이메일 분석

### 4. 탐지
- **PII 탐지**: 정규식 패턴 매칭
- **피싱 탐지**: 휴리스틱 알고리즘
- 결과가 데이터베이스에 저장됨

### 5. 대시보드
- 사용자가 결과 확인
- 그래프 및 통계
- 위험 평가

## 🔑 API Endpoints

### 인증
- `POST /api/v1/auth/register` - 회원가입
- `POST /api/v1/auth/login` - 로그인
- `GET /api/v1/auth/me` - 사용자 프로필

### Gmail OAuth
- `GET /api/v1/oauth/gmail/authorize` - Gmail 연결
- `GET /api/v1/oauth/gmail/callback` - OAuth 콜백

### 스캔
- `POST /api/v1/scans/` - 새 스캔 시작
- `GET /api/v1/scans/` - 스캔 목록
- `GET /api/v1/scans/{id}` - 스캔 상세 정보

### 탐지 결과
- `GET /api/v1/findings/` - 발견 항목 목록
- `GET /api/v1/findings/{id}` - 발견 항목 상세 정보
- `POST /api/v1/findings/{id}/resolve` - 발견 항목 해결

### 민감 정보
- `GET /api/v1/sensitive-items/` - 민감 정보 목록
- `POST /api/v1/sensitive-items/` - 민감 정보 추가
- `DELETE /api/v1/sensitive-items/{id}` - 민감 정보 삭제

### SafeShare
- `POST /api/v1/secure-files/upload-request` - 파일 업로드 요청
- `POST /api/v1/secure-files/upload-complete` - 파일 업로드 완료
- `POST /api/v1/secure-files/download-request` - 파일 다운로드 요청
