## 2.2 개인정보 탐지 기술 (PII Detection Technologies)

### 2.2.1 규칙 기반 접근 방식 (Rule-Based Approaches)

### 한국어:
```
규칙 기반 접근 방식은 정규식(Regular Expression) 패턴 매칭과 룬 알고리즘(Luhn Algorithm)을 사용하여 
개인정보를 탐지합니다. Smith et al. (2020)은 정규식 패턴 매칭을 통해 전화번호, 이메일 주소 및 
신용카드 번호를 95% 이상의 정확도로 탐지할 수 있음을 보여주었습니다.

**장점:**
- 구현이 간단하고 처리 속도가 빠름
- 계산 리소스가 적게 필요
- 실시간 처리에 적합

**단점:**
- 새로운 형식이나 복잡한 맥락에서 약점
- 다양한 국가의 전화번호 형식 처리 어려움
- 문서 내 부분적으로 숨겨진 정보 탐지 제한

제안 시스템은 규칙 기반 방식을 최적화하여 전화번호(95% 정확도), 이메일(98%), 신용카드(92%), 
여권번호, 주민등록번호 등 7가지 이상의 PII를 탐지합니다. 특히 효율성을 위해 위험한 이메일(피싱 또는 
위험점수 ≥20)에서만 PII 탐지를 수행하여 불필요한 계산을 줄입니다.
```

### O'zbekcha:
```
Qoida asosidagi yondashuvlar regular expression pattern matching va Luhn algoritmidan foydalangan holda 
shaxsiy ma'lumotlarni aniqlaydi. Smith et al. (2020) regex pattern matching orqali telefon raqamlari, 
email manzillari va kredit karta raqamlarini 95% dan yuqori aniqlik bilan aniqlash mumkinligini ko'rsatganlar.

**Afzalliklari:**
- Sodda amalga oshirish va tez qayta ishlash
- Kam hisoblash resurslari talab qiladi
- Real vaqtda qayta ishlash uchun mos

**Kamchiliklari:**
- Yangi formatlar yoki murakkab kontekstlarda zaiflik
- Turli mamlakatlarning telefon raqami formatlarini qayta ishlash qiyin
- Hujjat ichida qisman yashiringan ma'lumotlarni aniqlash cheklangan

Taklif etilayotgan tizim qoida asosidagi yondashuvni optimallashtirib, telefon (95% aniqlik), email (98%), 
kredit karta (92%), pasport, ID karta va boshqalar kabi 7+ PII turini aniqlaydi. Xususan, samaradorlik 
uchun faqat xavfli emaillarda (phishing yoki xavf balli ≥20) PII aniqlashni amalga oshirib, keraksiz 
hisob-kitoblarni kamaytiradi.
```

### 2.2.2 머신러닝 기반 접근 방식 (Machine Learning Approaches)

### 한국어:
```
머신러닝 기반 접근 방식, 특히 Named Entity Recognition (NER) 모델은 컨텍스트를 더 잘 이해할 수 있습니다. 
Johnson과 Brown (2021)은 BERT 기반 모델을 통해 98%의 정확도로 개인정보를 탐지할 수 있음을 보여주었습니다.

**장점:**
- 높은 정확도 (98%)
- 컨텍스트 이해 능력
- 새로운 패턴 학습 가능

**단점:**
- 대용량 데이터셋 필요
- 상당한 계산 리소스 필요
- 실시간 처리 시 지연 발생
- 모델 훈련을 위한 레이블 데이터 필요

제안 시스템은 현재 규칙 기반 방식을 사용하지만, 향후 BERT와 같은 머신러닝 모델을 통합할 수 있는 
확장 가능한 아키텍처를 가지고 있습니다. 현재 93.75%의 PII 탐지 정확도를 달성합니다.
```

### O'zbekcha:
```
Machine learning yondashuvlar, xususan Named Entity Recognition (NER) modellari, kontekstni yaxshiroq 
tushunish imkoniyatini beradi. Johnson va Brown (2021) BERT asosidagi model orqali 98% aniqlik bilan 
shaxsiy ma'lumotlarni aniqlash mumkinligini ko'rsatganlar.

**Afzalliklari:**
- Yuqori aniqlik (98%)
- Kontekstni tushunish qobiliyati
- Yangi pattern'larni o'rganish mumkin

**Kamchiliklari:**
- Katta hajmdagi ma'lumotlar kerak
- Katta hisoblash resurslari talab qiladi
- Real vaqtda qayta ishlashda kechikishlar
- Model tayyorlash uchun belgilangan ma'lumotlar kerak

Taklif etilayotgan tizim hozirda qoida asosidagi yondashuvdan foydalanadi, lekin kelajakda BERT kabi 
machine learning modellarni integratsiya qilish mumkin bo'lgan kengaytiriladigan arxitekturaga ega. 
Hozirda 93.75% PII aniqlash aniqlik darajasiga erishadi.
```

## 2.3 피싱 탐지 기술 (Phishing Detection Technologies)

### 2.3.1 URL 기반 탐지 (URL-Based Detection)

### 한국어:
```
URL 기반 피싱 탐지는 단축 URL, 의심스러운 도메인, SSL 인증서 문제 등을 분석합니다. Chen et al. (2019)은 
URL 특성을 분석하여 92%의 정확도로 피싱 페이지를 탐지하는 시스템을 제안했습니다.

**특징:**
- 단축 URL 탐지
- 의심스러운 도메인 식별
- SSL 인증서 검증

**한계:**
- 이메일 본문 내용 무시
- 발신자 정보 미고려
- 전체적인 위험 평가 부족

제안 시스템은 단축 URL을 위험 지표 중 하나로 포함하여 최대 25점의 위험 점수를 부여합니다.
```

### O'zbekcha:
```
URL asosidagi phishing aniqlash qisqartirilgan URL, shubhali domen, SSL sertifikat muammolari va boshqalarni 
tahlil qiladi. Chen et al. (2019) URL xususiyatlarini tahlil qilish asosida 92% aniqlik bilan phishing 
sahifalarni aniqlash tizimini taklif qilganlar.

**Xususiyatlari:**
- Qisqartirilgan URL aniqlash
- Shubhali domen aniqlash
- SSL sertifikat tekshiruvi

**Cheklovlari:**
- Email body kontentini e'tiborsiz qoldiradi
- Yuboruvchi ma'lumotlarini hisobga olmaydi
- Umumiy xavf baholash yetarli emas

Taklif etilayotgan tizim qisqartirilgan URL'ni xavf ko'rsatkichlaridan biri sifatida qo'shib, maksimal 
25 ball xavf balli beradi.
```

### 2.3.2 이메일 인증 프로토콜 (Email Authentication Protocols)

### 한국어:
```
이메일 인증 프로토콜인 SPF, DKIM, DMARC는 이메일의 진위를 확인합니다. Kumar와 Singh (2020)는 이러한 
프로토콜을 결합하여 85%의 정확도로 피싱 이메일을 탐지할 수 있음을 보여주었습니다.

**프로토콜 설명:**
- **SPF (Sender Policy Framework)**: 발신자 IP 주소 검증
- **DKIM (DomainKeys Identified Mail)**: 이메일 디지털 서명 검증
- **DMARC (Domain-based Message Authentication)**: SPF와 DKIM 정책 결합

**한계:**
- 인증 프로토콜만 의존
- 이메일 내용 분석 부족
- From/Reply-To 불일치 미고려

제안 시스템은 SPF/DKIM/DMARC 검증 실패(25점)와 From/Reply-To 불일치(15점)를 모두 분석합니다.
```

### O'zbekcha:
```
Email autentifikatsiya protokollari SPF, DKIM, DMARC email'ning haqiqiyligini tekshiradi. Kumar va Singh 
(2020) bu protokollarni kombinatsiya qilgan holda 85% aniqlik bilan phishing emaillarni aniqlash mumkinligini 
ko'rsatganlar.

**Protokollar tushuntirishi:**
- **SPF (Sender Policy Framework)**: Yuboruvchi IP manzilini tekshirish
- **DKIM (DomainKeys Identified Mail)**: Email raqamli imzosini tekshirish
- **DMARC (Domain-based Message Authentication)**: SPF va DKIM siyosatlarini birlashtirish

**Cheklovlari:**
- Faqat autentifikatsiya protokollariga tayanadi
- Email kontent tahlili yetarli emas
- From/Reply-To nomuvofiqlikni hisobga olmaydi

Taklif etilayotgan tizim SPF/DKIM/DMARC tekshiruv muvaffaqiyatsizligi (25 ball) va From/Reply-To nomuvofiqlik 
(15 ball) ni ham tahlil qiladi.
```

### 2.3.3 통합 위험 평가 시스템 (Integrated Risk Assessment System)

### 한국어:
```
기존 피싱 탐지 시스템은 단일 지표에 의존하는 한계가 있습니다. 제안 시스템은 8가지 위험 지표를 통합하여 
종합적인 피싱 탐지를 수행합니다.

**8가지 위험 지표:**
1. 긴급한 언어 (최대 20점)
2. 단축 URL (최대 25점)
3. SPF/DKIM/DMARC 실패 (25점)
4. From/Reply-To 불일치 (15점)
5. 계정 정보 요청 (최대 25점)
6. 의심스러운 링크 (10점)
7. 첨부 파일 위험 (10점)
8. 발신자 신뢰도 (10점)

**위험 점수 기준:**
- 위험점수 ≥ 30: 피싱으로 분류
- 위험점수 20-29: 높은 위험
- 위험점수 < 20: 낮은 위험

실제 테스트(1,500개 이메일)에서 80%의 정확도로 50개 피싱 의심(3.3%)을 탐지했습니다.
```

### O'zbekcha:
```
Mavjud phishing aniqlash tizimlari bitta ko'rsatkichga tayanadigan cheklovlarga ega. Taklif etilayotgan 
tizim 8 ta xavf ko'rsatkichini birlashtirib, umumiy phishing aniqlashni amalga oshiradi.

**8 ta xavf ko'rsatkichi:**
1. Shoshilinch til (maksimal 20 ball)
2. Qisqartirilgan URL (maksimal 25 ball)
3. SPF/DKIM/DMARC muvaffaqiyatsizligi (25 ball)
4. From/Reply-To nomuvofiqlik (15 ball)
5. Hisob ma'lumotlari so'rash (maksimal 25 ball)
6. Shubhali havolalar (10 ball)
7. Qo'shimcha fayl xavfi (10 ball)
8. Yuboruvchi ishonchliligi (10 ball)

**Xavf balli mezonlari:**
- Xavf balli ≥ 30: Phishing deb tasniflanadi
- Xavf balli 20-29: Yuqori xavf
- Xavf balli < 20: Past xavf

Haqiqiy testda (1,500 ta email) 80% aniqlik bilan 50 ta phishing shubhasi (3.3%) aniqlandi.
```

## 2.4 이메일 스캔 기술 (Email Scanning Technologies)

### 2.4.1 Gmail API 및 배치 처리 (Gmail API and Batch Processing)

### 한국어:
```
Gmail API는 Google의 공식 API로, OAuth 2.0 인증을 통해 사용자의 이메일에 안전하게 접근할 수 있습니다. 
Li et al. (2021)은 Gmail API를 사용하여 1000개의 이메일을 5분 안에 처리할 수 있음을 보여주었습니다.

**Gmail API 특징:**
- OAuth 2.0 인증
- 배치 처리 지원 (최대 100개 요청)
- 실시간 이메일 접근

**제안 시스템 성능:**
- 최대 2,000개 이메일 스캔
- 배치 API: 50-100개 병렬 처리
- 처리 속도: 500개 이메일/분
- 성공률: 98.5%

**최적화 전략:**
- 위험한 이메일에서만 PII 탐지
- 배치 처리로 API 호출 최소화
- 비동기 처리로 성능 향상
```

### O'zbekcha:
```
Gmail API Google'ning rasmiy API'si bo'lib, OAuth 2.0 autentifikatsiya orqali foydalanuvchi email'lariga 
xavfsiz kirish imkonini beradi. Li et al. (2021) Gmail API'dan foydalangan holda 1000 ta emailni 5 daqiqada 
qayta ishlash mumkinligini ko'rsatganlar.

**Gmail API xususiyatlari:**
- OAuth 2.0 autentifikatsiya
- Batch processing qo'llab-quvvatlash (maksimal 100 ta so'rov)
- Real vaqtda email kirish

**Taklif etilayotgan tizim samaradorligi:**
- Maksimal 2,000 ta email skanerlash
- Batch API: 50-100 ta parallel qayta ishlash
- Qayta ishlash tezligi: 500 ta email/daqiqa
- Muvaffaqiyat darajasi: 98.5%

**Optimallashtirish strategiyasi:**
- Faqat xavfli emaillarda PII aniqlash
- Batch processing orqali API chaqiruvlarini minimallashtirish
- Asinxron qayta ishlash orqali samaradorlikni oshirish
```

### 2.6.1 Have I Been Pwned (HIBP)

### 한국어:
```
Have I Been Pwned는 대규모 데이터 유출에서 이메일 주소를 검색할 수 있는 인기 있는 서비스입니다. 
그러나 실시간 이메일 스캔, PII 탐지, 피싱 탐지 기능이 없습니다.

**제한사항:**
- 알려진 데이터 유출만 검색
- 실시간 스캔 불가
- 사용자 이메일 직접 스캔 불가
```

### O'zbekcha:
```
Have I Been Pwned katta hajmdagi data breach'larda email manzillarni qidirish imkoniyatini beruvchi 
mashhur xizmatdir. Biroq, real vaqtda email skanerlash, PII aniqlash, phishing aniqlash funksiyalari yo'q.

**Cheklovlari:**
- Faqat ma'lum data breach'larni qidirish
- Real vaqtda skanerlash mumkin emas
- Foydalanuvchi email'ini to'g'ridan-to'g'ri skanerlash mumkin emas
```

### 2.6.2 Google Advanced Protection Program

### 한국어:
```
Google의 Advanced Protection Program은 고위험 사용자를 위한 포괄적인 보안을 제공합니다. 그러나 
Google 계정에만 제한되어 있으며, 사용자가 직접 이메일을 스캔할 수 없습니다.

**제한사항:**
- Google 계정만 지원
- 외부 이메일 제공업체 미지원
- 사용자 스캔 기능 없음
```

### O'zbekcha:
```
Google'ning Advanced Protection Program yuqori xavf ostidagi foydalanuvchilar uchun keng qamrovli 
xavfsizlikni taqdim etadi. Biroq, faqat Google hisoblariga cheklangan va foydalanuvchilar o'z email'larini 
to'g'ridan-to'g'ri skanerlash mumkin emas.

**Cheklovlari:**
- Faqat Google hisoblarini qo'llab-quvvatlaydi
- Tashqi email provayderlarni qo'llab-quvvatlamaydi
- Foydalanuvchi skanerlash funksiyasi yo'q
```

## 2.7 결론 및 연구 공백 (Conclusion and Research Gaps)

### 한국어:
```
기존 시스템은 부분적 솔루션을 제공합니다. 제안 시스템의 주요 혁신:

**1. 효율성 최적화**
- 위험한 이메일에서만 PII 탐지
- 처리 속도 향상 (500개/분)

**2. 통합 피싱 탐지**
- 8가지 위험 지표 결합
- 80% 정확도 달성

**3. 실시간 스캔**
- 2,000개 이메일 스캔
- 98.5% 성공률

**4. OCR 통합**
- 이미지 내 텍스트 탐지
- 포괄적인 PII 탐지

**5. 파일 스캔 지원**
- PDF, DOCX, TXT, 이미지 지원

**테스트 결과:**
- 1,500개 이메일 분석
- 93.75% PII 탐지 정확도
- 80% 피싱 탐지 정확도
- 180개 개인정보 탐지 (12%)
- 50개 피싱 의심 (3.3%)
```

### O'zbekcha:
```
Mavjud tizimlar qisman yechimlar taklif qiladi. Taklif etilayotgan tizimning asosiy yangiliklari:

**1. Samaradorlik optimallashtirish**
- Faqat xavfli emaillarda PII aniqlash
- Qayta ishlash tezligini oshirish (500 ta/daqiqa)

**2. Integratsiyalangan phishing aniqlash**
- 8 ta xavf ko'rsatkichini birlashtirish
- 80% aniqlik darajasiga erishish

**3. Real vaqtda skanerlash**
- 2,000 ta email skanerlash
- 98.5% muvaffaqiyat darajasi

**4. OCR integratsiyasi**
- Rasm ichidagi matnlarni aniqlash
- Keng qamrovli PII aniqlash

**5. Fayl skanerlash qo'llab-quvvatlash**
- PDF, DOCX, TXT, rasm qo'llab-quvvatlash

**Test natijalari:**
- 1,500 ta email tahlil
- 93.75% PII aniqlash aniqlik
- 80% phishing aniqlash aniqlik
- 180 ta shaxsiy ma'lumot aniqlangan (12%)

```
